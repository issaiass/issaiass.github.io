Robot Mapping with RTAB-Map – RoboND Map My World

RANGEL ALVARADO

January 1, 2026

RoboND_P4_MapMyWorld

GitHub Repository:
RoboND_P4_MapMyWorld

Brief Review

This repository contains all the necessary files to reproduce Project 4 (Map My World) from the Udacity Robotics Nanodegree. The goal of the project is to implement Simultaneous Localization and Mapping (SLAM) in a simulated environment using RTAB-Map, generating both a 2D occupancy grid and a 3D OctoMap.

The system uses a simulated differential-drive robot equipped with RGB-D and lidar sensors running in Gazebo, while RViz is used for visualization and analysis of the generated maps.

RTAB-Map (Real-Time Appearance-Based Mapping) is a widely adopted SLAM framework capable of real-time visual and visual-lidar SLAM, with strong loop-closure detection, efficient memory management, and robust long-term mapping capabilities.

One of RTAB-Map’s strengths is its extensive tooling and excellent ROS documentation, which makes it ideal for both learning and real-world robotic applications. Mastering RTAB-Map provides a solid foundation for advanced topics such as autonomous navigation and long-term localization.

This project leverages the rtabmap_ros package, which acts as a ROS wrapper around RTAB-Map’s core C++ library.

Project Overview

The repository is organized into multiple ROS packages to clearly separate concerns:

  • my_robot – Robot description, sensors, Gazebo world, and launch files
  • rtabmap_pkg – Teleoperation, mapping, and localization pipelines

Using the Package

Clone and build the workspace:

cd ~/catkin_ws/src
git clone https://github.com/issaiass/RoboND_P4_MapMyWorld.git
cd ..
catkin_make
source devel/setup.bash

Running Mapping

roslaunch my_robot world.launch
roslaunch rtabmap_pkg teleop.launch
roslaunch rtabmap_pkg mapping.launch

Localization Mode

Disable incremental memory and reuse the generated database:

roslaunch rtabmap_pkg localization.launch

Best Practices

  • Use low velocities for better sensor fusion
  • Perform multiple loop closures
  • Backup the RTAB-Map database before remapping

Future Work

  • Autonomous navigation integration
  • Deployment on physical robots
  • Multi-session mapping
